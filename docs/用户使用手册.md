## 1. 系统概述

### 1.1 关于SwiftAdmin

SwiftAdmin 是一套基于 ThinkPHP 6.0 后端框架与 Vue 3 前端框架开发的企业级后台管理系统。系统采用前后端分离架构，前端基于 TypeScript，使用 Naive UI 组件库与 ECharts 可视化图表库构建界面。适用于中小型企业快速搭建后台管理平台，支持二次开发与模块扩展。

| 层级 | 技术 |
|------|------|
| 后端框架 | ThinkPHP 6.0（PHP 7.2+，兼容 PHP 8.1） |
| 前端框架 | Vue 3 + TypeScript |
| UI组件库 | Naive UI |
| 图表库 | ECharts |
| 数据库 | MySQL |
| 前端构建 | Vite |

### 1.2 浏览器兼容性

推荐使用最新版本的 Chrome、Edge 或 Firefox 浏览器。系统界面采用现代化设计，建议屏幕分辨率不低于 1366x768。

### 1.3 系统访问

部署完成后，通过浏览器访问系统地址，系统会自动跳转至登录页面。默认管理员账号如下：

- 用户名：`admin`
- 密码：`admin123`

> 安全提示：首次登录后，建议立即修改默认密码。

## 2. 登录系统

### 2.1 登录页面

访问系统地址后，浏览器将显示登录页面。页面背景为渐变紫色调，中央显示登录卡片。

**登录界面元素：**

- 用户名输入框：输入管理员账号
- 密码输入框：输入登录密码（支持回车键提交）
- 登录按钮：点击或按回车键提交登录请求

### 2.2 登录流程

1. 在用户名输入框中输入管理员账号（默认：admin）
2. 在密码输入框中输入密码（默认：admin123）
3. 点击【登录】按钮或按回车键
4. 验证通过后，系统将自动跳转至控制台页面

### 2.3 退出登录

登录成功后，页面右上角显示当前用户头像与昵称。点击用户区域的向下箭头，在弹出的下拉菜单中选择【退出登录】即可安全退出系统。

## 3. 控制台（Dashboard）

控制台是登录成功后的默认展示页面，提供系统运行概览。

### 3.1 欢迎区域

页面顶部显示欢迎语"欢迎回来，[管理员昵称]"以及当前日期。副标题显示"SwiftAdmin 控制台 -- 系统运行概览"。

### 3.2 统计卡片

欢迎区域下方显示四张统计卡片，展示系统关键指标。每张卡片包含图标、数值和标签说明，底部有对应颜色的指示条。典型统计指标包括用户总数、文章总数等。

### 3.3 图表区域

页面中部并排显示两个折线图面板：

- **用户注册趋势（近7天）：** 紫色主题折线图，展示最近7天每日新增用户数
- **文章发布趋势（近7天）：** 蓝色主题折线图，展示最近7天每日文章发布量

### 3.4 底部信息区

- **左侧 -- 文章分类饼图：** 以饼图形式展示文章在各分类下的分布情况
- **右侧 -- 最近操作：** 列表形式展示系统最近的操作日志，包含操作人员、操作标题和操作时间

### 3.5 快速导航

通过左侧菜单栏可快速跳转至其他功能模块。顶部标签页支持多页面同时打开，方便在不同功能之间快速切换。

## 4. 权限管理

权限管理模块包含管理员管理、角色管理、菜单规则和管理员日志四个子模块，是系统访问控制的核心。

### 4.1 管理员管理

**路径：** 权限管理 > 管理员管理

**功能说明：** 管理后台管理员账号，支持新增、编辑、删除管理员，可设置管理员所属角色。

**主要操作：**

- **新增管理员：** 点击列表上方【新增】按钮，在弹出的表单中填写用户名、密码、昵称，选择所属角色组，上传头像（可选），点击确认完成创建
- **编辑管理员：** 点击列表中某行的【编辑】按钮，修改管理员信息（密码留空表示不修改密码）
- **删除管理员：** 点击【删除】按钮，确认后移除管理员账号
- **状态切换：** 可通过状态开关快速启用或禁用管理员账号
- **搜索筛选：** 列表顶部提供搜索框，支持按用户名或昵称进行模糊搜索

**注意事项：**

- 不能删除当前登录的管理员自身
- 删除操作不可恢复，请谨慎操作
- 禁用状态的管理员无法登录后台

### 4.2 角色管理

**路径：** 权限管理 > 角色管理

**功能说明：** 管理系统角色组，每个角色定义一组权限规则。通过为管理员分配角色，实现细粒度的权限控制。

**主要操作：**

- **新增角色：** 点击【新增】按钮，输入角色名称，在权限树中勾选该角色拥有的菜单权限和操作权限
- **编辑角色：** 点击【编辑】按钮，修改角色名称和权限分配
- **删除角色：** 删除不再使用的角色（需确保没有管理员正在使用该角色）
- **权限分配：** 权限树以层级结构展示所有菜单与操作权限，支持全选和逐项勾选

**权限说明：**

- 一级节点对应侧边栏主菜单的可见性
- 二级节点对应子菜单页面的访问权限
- 每个页面可细分为查看、新增、编辑、删除等操作权限

### 4.3 菜单规则

**路径：** 权限管理 > 菜单规则

**功能说明：** 管理左侧导航菜单的层级结构、显示顺序和权限标识。菜单规则是系统权限控制的基础数据。

**主要操作：**

- **新增菜单：** 选择父级菜单（一级菜单选择顶级），填写菜单名称、路由路径、组件路径、图标、排序号和权限标识
- **编辑菜单：** 修改菜单的各项属性，修改后需刷新页面方可生效
- **删除菜单：** 删除菜单及其关联的子菜单和权限标识
- **排序：** 通过排序号控制同级菜单的显示先后顺序，数字越小越靠前
- **类型区分：** 支持设置菜单类型（目录/菜单/按钮），目录类型仅作为分组，不渲染页面

**重要参数说明：**

- **菜单名称（title）：** 侧边栏显示的文字
- **路由路径（path）：** 前端路由地址，如 `/auth/admin`
- **组件路径（component）：** 前端 Vue 组件文件的相对路径
- **权限标识（permission）：** 后端权限验证标识，如 `auth:admin:list`
- **图标（icon）：** 菜单图标名称，使用 Ionicons 图标集

### 4.4 管理员日志

**路径：** 权限管理 > 管理员日志

**功能说明：** 记录所有管理员的操作行为，用于安全审计和问题追溯。

**日志记录内容：**

- 操作管理员用户名
- 操作标题（方法说明）
- 请求URL地址
- 请求方法（GET/POST/PUT/DELETE）
- 客户端IP地址与User-Agent
- 请求参数（脱敏处理）
- 操作耗时（秒）
- 操作结果状态

**主要操作：**

- **查看日志详情：** 点击某条日志查看完整的请求参数和响应信息
- **搜索过滤：** 支持按管理员名称、操作标题、时间范围进行筛选
- **分页浏览：** 日志列表支持分页，默认按操作时间倒序排列
- **定期清理：** 可配合定时任务配置日志自动清理策略

## 5. 会员管理

### 5.1 会员列表

**路径：** 会员管理 > 会员列表

**功能说明：** 管理前端注册的会员用户，支持查看、编辑、禁用、删除等操作。

**会员信息字段：** 会员ID、头像、用户名/昵称、手机号、邮箱、注册时间、登录IP、登录次数、最后登录时间、账户状态（正常/禁用）、所属会员分组。

**主要操作：**

- **查看详情：** 点击某位会员行可查看详细信息
- **编辑会员：** 修改会员基本资料、调整分组
- **禁用/启用：** 通过状态开关控制会员账号是否可用
- **删除会员：** 删除会员账号（通常为软删除，数据可恢复）
- **搜索筛选：** 支持按用户名、手机号、邮箱、注册时间范围等条件进行筛选

### 5.2 会员分组

**路径：** 会员管理 > 会员分组

**功能说明：** 管理会员分组类别，用于对会员进行分类管理和差异化权限控制。

**主要操作：**

- **新增分组：** 创建新的会员分组，设置分组名称
- **编辑分组：** 修改分组名称
- **删除分组：** 删除不再使用的分组（需确保该分组下无会员）
- **分组应用：** 在会员编辑页面为会员指定所属分组

## 6. 系统管理

### 6.1 系统配置

**路径：** 系统管理 > 系统配置

**功能说明：** 管理系统全局参数配置，包括站点信息、上传设置、邮件配置等。

**配置分组：**

- **站点设置：** 站点名称、站点LOGO、备案号等
- **上传设置：** 上传驱动（本地/云存储）、最大上传大小、允许的文件格式
- **其他设置：** 根据系统扩展可包含邮件、短信、支付等第三方服务配置

**主要操作：**

- **修改配置：** 在对应的配置项输入框中修改值，点击保存后即刻生效
- **上传LOGO：** 点击LOGO配置项旁的上传按钮，选择图片文件上传

**默认配置项说明：**

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| site_name | SwiftAdmin | 站点名称，显示在浏览器标题栏 |
| site_logo | （空） | 站点LOGO图片URL |
| site_icp | （空） | ICP备案号 |
| upload_driver | local | 上传驱动类型 |
| upload_max_size | 10485760（10MB） | 文件上传最大字节数 |
| upload_allowed_ext | jpg,jpeg,png,gif,doc,docx,pdf,zip | 允许上传的文件格式 |

### 6.2 附件管理

**路径：** 系统管理 > 附件管理

**功能说明：** 管理系统中所有上传的文件附件，支持查看、下载、删除操作。

**附件列表显示信息：** 文件名（原始文件名）、文件大小、上传时间、上传者（管理员或会员）、存储路径。

**主要操作：**

- **上传附件：** 点击【上传】按钮，选择本地文件上传
- **查看附件：** 图片类型附件支持在线预览
- **下载附件：** 点击下载按钮获取原始文件
- **删除附件：** 删除附件文件（删除后文件从存储中移除）
- **批量删除：** 勾选多条附件后执行批量删除
- **搜索：** 支持按文件名模糊搜索

## 7. 数据管理

数据管理模块负责数据安全和回收机制，包含四个子模块。

### 7.1 数据回收站

**路径：** 数据管理 > 数据回收站

**功能说明：** 统一管理被软删除的数据记录。系统对删除操作默认采用软删除机制，被删除数据首先进入回收站，可在回收站中恢复或彻底删除。

**主要操作：**

- **恢复数据：** 选择回收站中的记录，点击【恢复】按钮将数据恢复到原始表中
- **彻底删除：** 从回收站中永久删除数据，此操作不可逆
- **清空回收站：** 一键清空所有回收站数据
- **搜索筛选：** 支持按数据类型、删除时间等条件筛选

### 7.2 敏感数据修改记录

**路径：** 数据管理 > 敏感数据修改记录

**功能说明：** 记录系统中敏感字段（如手机号、邮箱、余额等）的所有修改操作，用于合规审计。

**日志内容：** 修改时间、操作人、所属数据表、修改字段名称、修改前值、修改后值。

**主要操作：**

- **查看记录：** 浏览所有敏感数据的变更历史
- **搜索筛选：** 按操作人、数据表、时间范围等条件筛选
- **详情查看：** 点击记录查看完整的字段变更详情

### 7.3 数据回收规则管理

**路径：** 数据管理 > 数据回收规则管理

**功能说明：** 配置哪些数据表启用软删除回收机制。通过配置回收规则，可精准控制回收站管理的目标数据表。

**主要操作：**

- **新增规则：** 选择目标数据表，配置回收规则
- **编辑规则：** 修改已有规则的数据表和参数
- **删除规则：** 移除不再需要的回收规则
- **启用/禁用：** 通过状态开关控制规则是否生效

### 7.4 敏感字段规则管理

**路径：** 数据管理 > 敏感字段规则管理

**功能说明：** 定义哪些数据表中的哪些字段属于敏感字段。配置后，对这些字段的修改操作将自动记录到敏感数据修改记录中。

**主要操作：**

- **新增规则：** 选择数据表和字段，标记为敏感字段
- **编辑规则：** 修改敏感字段规则配置
- **删除规则：** 移除敏感字段标记
- **启用/禁用：** 控制规则是否启用监控

## 8. 代码生成

**路径：** 代码生成

**功能说明：** 可视化代码生成器，通过配置数据表和字段信息，一键生成完整的 CRUD（增删改查）前后端代码。

**操作流程：**

1. **选择数据表：** 在数据表列表中选择需要生成代码的目标表
2. **配置字段：** 为每个字段配置显示类型（文本框、下拉框、图片上传、开关、富文本等）、验证规则、搜索方式
3. **配置关联：** 设置外键关联、下拉数据源等
4. **生成代码：** 点击【生成】按钮，系统自动生成控制器、模型、验证器、前端页面等完整代码文件
5. **添加菜单：** 自动在菜单规则中创建对应的菜单项

**支持的字段类型：** input（文本框）、number（数字框）、select（下拉框）、switch（开关）、image（图片上传）、file（文件上传）、editor（富文本编辑器）、date（日期选择器）、textarea（文本域）、radio/checkbox（单选/多选）。

## 9. CMS内容管理

### 9.1 栏目管理

**路径：** CMS > 栏目管理

**功能说明：** 管理网站的栏目分类结构，支持无限级分类。

**主要操作：**

- **新增栏目：** 选择父级栏目，填写栏目名称、排序号等
- **编辑栏目：** 修改栏目名称、排序等信息
- **删除栏目：** 删除栏目（如栏目下有文章则不可删除）
- **排序：** 通过修改排序号调整栏目显示顺序

### 9.2 文章管理

**路径：** CMS > 文章管理

**功能说明：** 管理网站的文章内容，支持文章的发布、编辑、删除等操作。

**文章字段说明：** 文章标题、所属栏目、封面图片、文章摘要、文章正文（支持富文本编辑）、发布状态（草稿/已发布）、发布时间、排序号、SEO标题、关键词、描述。

**主要操作：**

- **新增文章：** 填写文章信息，使用富文本编辑器编写正文，可插入图片、视频等媒体内容
- **编辑文章：** 修改已有文章的内容和属性
- **删除文章：** 将文章移入回收站（软删除）
- **发布/下架：** 控制文章在前端是否可见
- **搜索筛选：** 按标题、栏目、状态、发布时间等条件筛选

## 10. AI智能开发

**路径：** AI智能开发

**功能说明：** 集成AI大模型能力，通过自然语言描述需求，辅助生成代码、API接口和数据库设计。

**主要功能：**

- **需求输入：** 在对话框中用自然语言描述开发需求
- **代码生成：** AI根据需求生成对应的控制器、模型、前端页面代码
- **对话记录：** 保留与AI的对话历史，方便回溯和迭代
- **代码预览：** 生成的代码可在线预览，确认后可一键应用到项目中

**使用建议：**

- 需求描述应尽量具体，包含数据表结构、业务逻辑等关键信息
- 生成的代码建议先检查再应用，确保符合项目规范
- 可结合代码生成器模块，对比使用效果更佳

## 11. 数据大屏

**路径：** 数据大屏

**功能说明：** 通过 ECharts 可视化图表展示全屏数据看板，适用于大屏幕展示和数据分析汇报。

**页面特征：**

- 全屏显示模式，无侧边栏和顶栏
- 深色背景，数据可视化效果突出
- 多种图表类型组合展示（柱状图、折线图、饼图等）
- 实时数据更新

**适用场景：** 经营数据分析汇报、系统运行监控展示、会议大屏演示。

## 15. 定时任务

**路径：** 定时任务

**功能说明：** 管理系统中的计划任务，支持创建、编辑、执行和监控定时任务。

**任务类型：** URL请求任务、Shell命令任务、PHP类方法任务。

**主要操作：**

- **新增任务：** 设置任务名称、执行规则（Cron表达式）、任务类型和具体执行内容
- **编辑任务：** 修改任务配置
- **手动执行：** 点击执行按钮立即触发一次任务
- **启用/禁用：** 控制任务是否按计划执行
- **查看日志：** 查看任务的执行历史和执行结果

**Cron表达式格式（5位）：**

```
分钟 小时 日期 月份 星期
*    *    *    *    *
```

常用示例：

| 说明 | 表达式 |
|------|--------|
| 每分钟执行 | `* * * * *` |
| 每小时执行 | `0 * * * *` |
| 每天凌晨2点 | `0 2 * * *` |
| 每周一上午9点 | `0 9 * * 1` |

## 22. 常见问题（FAQ）

### Q1：忘记管理员密码怎么办？

可通过数据库中直接修改。登录 MySQL 数据库管理工具，找到 `swift_admin` 表，使用系统密码加密方式生成新密码哈希值后更新 `password` 字段。或在项目根目录执行密码重置命令（如系统提供命令行工具）。

### Q2：登录后页面空白或加载异常？

可能原因及解决方法：

- **前端资源未正确构建：** 进入 `frontend/admin` 目录，执行 `npm run build` 重新构建前端资源
- **浏览器缓存问题：** 清除浏览器缓存后强制刷新页面（Ctrl+F5）
- **路由权限问题：** 检查数据库 `swift_menu` 表中菜单数据是否完整
- **后端API异常：** 检查 PHP 错误日志确认是否存在后端报错

### Q3：如何添加新的功能菜单？

1. 进入【权限管理 > 菜单规则】
2. 点击【新增】按钮
3. 选择父级菜单（一级菜单选顶级）
4. 填写菜单名称、路由路径、组件路径、权限标识
5. 设置排序号和图标
6. 保存后在【角色管理】中将该菜单权限分配给对应角色
7. 刷新页面后菜单即可在侧边栏显示

### Q4：上传文件失败怎么办？

依次检查以下各项：

1. **文件大小：** 确认文件未超过系统配置中 `upload_max_size` 的限制（默认10MB）
2. **文件格式：** 确认文件后缀名在 `upload_allowed_ext` 允许格式列表中
3. **目录权限：** 确认 `public/uploads` 目录具有写入权限（Linux需设为755或777）
4. **PHP配置：** 检查 `php.ini` 中的 `upload_max_filesize` 和 `post_max_size` 是否足够大

### Q5：如何备份系统数据？

建议同时备份以下内容：

1. **数据库：** 使用 mysqldump 或数据库管理工具导出完整的SQL文件
2. **上传文件：** 备份 `public/uploads` 目录下的所有附件文件
3. **配置文件：** 备份项目根目录下的 `.env` 文件
4. **前端构建产物：** 保留 `public` 目录下的前端编译文件

### Q6：系统运行缓慢如何优化？

- 开启数据库查询缓存
- 开启 PHP OPcache 加速
- 对频繁查询的数据表添加合适的索引
- 定期清理管理员日志和回收站中的过期数据
- 将静态资源部署到CDN
- 启用 Redis 作为缓存和Session驱动

### Q7：如何开启调试模式？

在项目根目录的 `.env` 文件中，将 `APP_DEBUG` 设置为 `true`。调试模式开启后，页面会显示详细的错误信息和SQL日志，方便开发调试。生产环境务必关闭调试模式。

### Q8：支持哪些PHP版本？

系统要求 PHP 7.2 及以上版本，兼容 PHP 8.1。推荐使用 PHP 7.4 或 PHP 8.0 以获得最佳性能和兼容性。

### Q9：如何对接第三方登录（微信、QQ等）？

可通过模块市场安装对应的第三方登录模块，或自行在 `modules` 目录下开发扩展模块进行对接。需要配置对应的 AppID、AppSecret 等参数。

### Q10：定时任务不自动执行怎么办？

1. 确认服务器已配置系统级 Cron 任务指向项目的定时任务入口脚本
2. 检查定时任务管理页面中该任务是否处于【启用】状态
3. 查看任务执行日志确认是否有报错信息
4. 可使用【手动执行】按钮测试任务是否可正常执行

### Q11：侧边栏颜色修改后如何恢复默认？

在布局配置面板中点击底部的【恢复默认】按钮，所有自定义的侧边栏、顶栏和布局设置将一并重置为系统出厂默认值。

### Q12：模块市场安装模块失败怎么办？

- 检查服务器网络是否可正常访问模块市场源站
- 确认 `modules` 目录具有写入权限
- 检查 PHP 是否启用了 `file_get_contents` 或 `curl` 扩展
- 查看系统日志了解具体错误原因
