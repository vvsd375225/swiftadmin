## 1. 环境要求

### 1.1 运行环境

| 依赖 | 最低版本 | 推荐版本 | 说明 |
|------|---------|---------|------|
| **PHP** | 7.4 | 8.1+ | ThinkPHP 6.0 要求 PHP >= 7.2.5 |
| **PHP 扩展** | pdo_mysql, gd, fileinfo, zip | 另需 redis, opcache | 生产环境必须安装 opcache |
| **MySQL** | 5.7 | 8.0+ | 字符集 utf8mb4 |
| **Redis** | 5.0 | 7.0+ | 缓存 / Session / 队列 |
| **Node.js** | 16 | 20 LTS | 前端构建用 |
| **Nginx** | 1.18 | 1.24+ | 或 Apache 2.4+ |
| **Composer** | 2.x | 最新稳定版 | PHP 依赖管理 |

### 1.2 PHP 扩展检查

```bash
# Linux / macOS
php -m | grep -E "pdo_mysql|fileinfo|gd|zip|redis"

# Windows
php -m | findstr /i "pdo_mysql fileinfo gd zip redis"
```

缺少扩展时请修改 `php.ini` 取消注释或通过包管理器安装。

### 1.3 项目结构

```
项目根目录/
├── app/admin/             # 后端应用（ThinkPHP 6.0 多应用模式）
├── config/                # 框架配置文件（database.php, cache.php 等）
├── database/install.sql   # 数据库初始化脚本
├── frontend/admin/        # 前端管理后台（Vue 3 + Naive UI + Vite）
├── public/                # Web 入口目录（index.php）
│   ├── admin/             # 前端构建产物输出到此
│   ├── uploads/           # 上传文件目录
│   └── static/            # 静态资源目录
├── runtime/               # 运行时目录（缓存、日志）
├── vendor/                # Composer 依赖
├── .env                   # 环境配置文件（数据库、Redis 等）
├── install.sh             # Linux/macOS 一键安装脚本
├── install.bat            # Windows 一键安装脚本
├── install-bt.sh          # 宝塔面板安装脚本
├── Dockerfile             # Docker 镜像构建文件
├── docker-compose.yml     # Docker 编排文件
└── nginx.conf             # Docker 环境 Nginx 配置
```

## 2. 方式一：一键安装 Linux/macOS

适合 **Linux** 或 **macOS** 环境，已安装好 PHP、Composer、Node.js、MySQL 的场景。

### 2.1 执行安装

```bash
# 进入项目目录
cd /path/to/SwiftAdmin

# 赋予执行权限
chmod +x install.sh

# 运行安装脚本
bash install.sh
```

### 2.2 脚本执行流程

脚本按以下 7 个步骤依次执行：

| 步骤 | 操作 | 说明 |
|------|------|------|
| **[1/7]** | 环境检测 | 检查 PHP / Composer / Node / npm 是否可用；验证 PHP >= 7.2.5、Node >= 16；检查 pdo_mysql、gd、fileinfo、zip 扩展 |
| **[2/7]** | 数据库配置 | 交互式输入：主机(默认127.0.0.1)、端口(3306)、用户名(root)、密码、库名(swiftadmin)、表前缀(swift_) |
| **[3/7]** | 创建数据库 | 执行 `CREATE DATABASE IF NOT EXISTS`，字符集 utf8mb4 |
| **[4/7]** | 导入表结构 | 将 `database/install.sql` 导入目标数据库 |
| **[5/7]** | 生成 .env | 根据第 2 步的配置自动生成 `.env` 文件，包含 [DATABASE]、[REDIS]、[CACHE] 三个配置段 |
| **[6/7]** | 安装依赖 | 执行 `composer install` + `npm install` + `npm run build` 构建前端 |
| **[7/7]** | 设置权限 | 创建 runtime、public/uploads、public/static 目录并设为 777 权限 |

### 2.3 安装完成后

```bash
# 启动 PHP 内置服务器（开发用）
php think run

# 启动前端开发服务器（可选，开发时使用）
cd frontend/admin && npm run dev
```

访问后台：**`http://localhost:8000/admin`**（使用 `php think run` 时默认端口 8000）

## 3. 方式二：一键安装 Windows

适合 Windows 环境，已安装 PHP、Composer、Node.js、MySQL 并配置好环境变量。

### 3.1 前提条件

确保以下命令在 CMD 或 PowerShell 中可执行：

```cmd
php --version
composer --version
node --version
npm --version
mysql --version
```

推荐使用 [phpstudy_pro](https://www.xp.cn/) 集成环境，一键搞定 PHP + MySQL + Nginx。

### 3.2 执行安装

双击运行 `install.bat`，或在命令行中执行：

```cmd
cd C:\path\to\SwiftAdmin
install.bat
```

### 3.3 脚本执行流程

与 Linux 版脚本完全对应，同样 7 个步骤，区别在于：

- 使用 `set /p` 替代 `read` 进行交互输入
- 权限步骤不再执行 `chmod`，改为创建目录（Windows 无需额外权限设置）
- 末尾 `pause` 保持窗口不关闭，方便查看结果

### 3.4 安装完成后

```cmd
:: 启动 PHP 内置服务器
php think run

:: 或使用 Nginx / Apache 配置站点根目录到 public/
```

访问后台：**`http://localhost:8000/admin`**

## 4. 方式三：Docker 部署

Docker 部署将后端、前端、MySQL、Redis、Nginx 全部容器化，一次启动全部服务。

### 4.1 前置条件

- Docker >= 20.10
- Docker Compose >= 2.0

### 4.2 Docker 架构说明

| 容器 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| **nginx** | nginx:1.25-alpine | 80 | 反向代理 + 静态资源，对接 php-fpm |
| **app** | 项目构建 (php:8.0-fpm) | 9000 | PHP-FPM 处理动态请求 |
| **mysql** | mysql:5.7 | 3306 | 数据库，自动导入 install.sql |
| **redis** | redis:7-alpine | 6379 | 缓存服务 |

### 4.3 启动服务

```bash
# 进入项目目录
cd /path/to/SwiftAdmin

# 构建镜像并启动所有容器（后台运行）
docker-compose up -d --build

# 查看容器运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 4.4 工作流程

1. MySQL 容器启动后自动执行 `database/install.sql` 初始化表结构
2. App 容器构建时执行 `composer install --no-dev` + 前端 `npm run build`
3. `.env.docker` 作为环境配置文件挂载到容器内，**数据库主机名填写 `mysql`、Redis 主机名填写 `redis`**（Docker 网络内部 DNS）
4. Nginx 根据 `nginx.conf` 转发 PHP 请求到 app:9000

### 4.5 自定义配置

```bash
# 通过环境变量覆盖默认值
NGINX_PORT=8080 DB_PASSWORD=mypass123 docker-compose up -d

# 或在 .env.docker 中修改配置后重建
docker-compose up -d --build
```

### 4.6 常用管理命令

```bash
# 停止服务
docker-compose down

# 停止并删除数据卷（慎用，会清除数据库数据）
docker-compose down -v

# 重启单个服务
docker-compose restart app

# 进入容器内部
docker exec -it swiftadmin-app bash
```

## 5. 方式四：宝塔面板 CentOS

适用于已安装**宝塔面板**的 CentOS / Alibaba Cloud Linux / AlmaLinux 服务器。

### 5.1 宝塔面板环境准备

在宝塔面板 **软件商店** 中安装以下软件：

- **PHP 7.4 及以上**（推荐 PHP 8.1）
- **MySQL 5.7 及以上**
- **Redis**（可选，推荐）
- **Nginx**（推荐）或 Apache
- **Node.js 版本管理器**（可选，需要前端构建时安装）

### 5.2 执行安装脚本

```bash
# 进入项目目录
cd /www/wwwroot/SwiftAdmin

# 赋予执行权限
chmod +x install-bt.sh

# 运行宝塔面板专用安装脚本
bash install-bt.sh
```

### 5.3 脚本执行流程

| 步骤 | 操作 | 说明 |
|------|------|------|
| **[1/9]** | 宝塔面板检测 | 检测 `/www/server/panel` 中 bt 命令是否存在 |
| **[2/9]** | PHP 检测 | 自动扫描 /www/server/php/ 下 74/80/81/82/83 版本，选第一个找到的 |
| **[3/9]** | Composer 设置 | 如不存在则自动安装到 /usr/local/bin/composer |
| **[4/9]** | Node.js 检测 | 检测 node/npm，未安装时提示（可跳过） |
| **[5/9]** | 数据库配置 | 交互式输入数据库信息 |
| **[6/9]** | 创建数据库 | `CREATE DATABASE IF NOT EXISTS` |
| **[7/9]** | 导入表结构 | 执行 install.sql |
| **[8/9]** | 生成 .env | 自动生成环境配置文件 |
| **[9/9]** | 安装依赖与权限 | composer install --no-dev，npm run build，设置 777 权限和 www 属主 |

### 5.4 手动完成站点配置

脚本运行完毕后，还需要在宝塔面板中手动设置站点：

#### 步骤一：添加站点

宝塔面板 > 网站 > 添加站点：

- **域名**: 填写你的域名或 IP
- **根目录**: 选择项目目录下的 **`public`** 子目录
- **PHP 版本**: 选择 PHP 7.4+
- 其他保持默认，点击提交

#### 步骤二：设置伪静态

站点设置 > 伪静态，粘贴以下配置：

```nginx
location / {
    if (!-e $request_filename) {
        rewrite ^(.*)$ /index.php?s=$1 last;
    }
}
```

#### 步骤三：配置 Redis（可选）

- 宝塔软件商店 > Redis > 安装并启动
- PHP 设置 > 安装扩展 > 勾选 redis 扩展
- 在 `.env` 中配置 Redis 密码（如设置了密码）

### 5.5 访问与登录

```
后台地址：http://你的域名/admin
默认账号：admin
默认密码：admin123
```

## 6. 方式五：手动安装

如果你需要精细控制每一步，或环境特殊不适合用脚本，可以按以下步骤手动安装。

### 6.1 克隆 / 上传项目

```bash
# 克隆项目（或上传、解压到目标目录）
cd /www/wwwroot
# 将项目文件放到此处，例如 /www/wwwroot/SwiftAdmin
```

### 6.2 安装 PHP 依赖

```bash
cd /www/wwwroot/SwiftAdmin
composer install --no-dev --optimize-autoloader
```

如果服务器无法访问外网，可以先在本地执行 `composer install`，然后将整个 `vendor/` 目录一并上传。

### 6.3 配置数据库

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE IF NOT EXISTS swiftadmin DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 导入表结构
USE swiftadmin;
SOURCE /www/wwwroot/SwiftAdmin/database/install.sql;

-- 退出
EXIT;
```

### 6.4 配置 .env 文件

在项目根目录创建 `.env` 文件：

```ini
APP_DEBUG = true

[DATABASE]
TYPE = mysql
HOSTNAME = 127.0.0.1
DATABASE = swiftadmin
USERNAME = root
PASSWORD = your_password
HOSTPORT = 3306
CHARSET = utf8mb4
PREFIX = swift_

[REDIS]
HOST = 127.0.0.1
PORT = 6379
PASSWORD =
SELECT = 0

[CACHE]
DRIVER = file
```

请将 `your_password` 替换为实际的数据库密码。

### 6.5 构建前端

```bash
cd frontend/admin

# 安装 Node.js 依赖
npm install

# 构建生产版本（输出到 public/admin/）
npm run build
```

如果服务器没有 Node.js，可以在本地开发机执行 `npm run build`，然后将 `frontend/admin/dist/` 目录下的内容上传到服务器 `public/admin/` 目录。

### 6.6 设置目录权限

```bash
# 创建必要目录
mkdir -p runtime public/uploads public/static

# Linux / macOS 权限设置
chmod -R 777 runtime public/uploads public/static

# 宝塔面板额外设置属主
chown -R www:www /www/wwwroot/SwiftAdmin
```

### 6.7 配置 Web 服务器

参见下面 [Nginx 配置参考](#8-nginx-配置参考) 章节。

## 7. 部署后配置

### 7.1 默认登录信息

| 项目 | 值 |
|------|-----|
| 后台地址 | `http://你的域名/admin` |
| 管理员账号 | `admin` |
| 管理员密码 | `admin123` |

> **安全提醒**: 首次登录成功后，请立即修改默认密码！

### 7.2 快速验证

访问以下地址确认系统正常运行：

```
# 检查 PHP 是否正常解析
http://你的域名/

# 进入后台登录页
http://你的域名/admin
```

### 7.3 关闭调试模式

生产环境务必关闭调试模式，修改 `.env`：

```ini
APP_DEBUG = false
```

### 7.4 配置 Redis（推荐）

编辑 `.env`，将缓存驱动从 `file` 改为 `redis`：

```ini
[REDIS]
HOST = 127.0.0.1
PORT = 6379
PASSWORD = your_redis_password
SELECT = 0

[CACHE]
DRIVER = redis
```

确保 PHP 已安装 redis 扩展：

```bash
php -m | grep redis
```

### 7.5 重置管理员密码

如果忘记密码，可以通过以下 SQL 重置：

```sql
-- ThinkPHP 框架默认使用 MD5 + Salt 方式加密
-- 以下为重置为 admin123 的示例（盐值请根据实际加密逻辑调整）
UPDATE swift_admin SET password = MD5(CONCAT(MD5('admin123'), salt)) WHERE username = 'admin';
```

或者通过 ThinkPHP 命令行：

```bash
php think admin:reset-password admin admin123
```

## 8. Nginx 配置参考

### 8.1 标准配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /www/wwwroot/SwiftAdmin/public;
    index index.php index.html;

    charset utf-8;
    client_max_body_size 50m;

    # ThinkPHP URL 重写：静态文件直接返回，其余交 index.php 处理
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # PHP 请求转发
    location ~ \.php$ {
        fastcgi_pass 127.0.0.1:9000;          # 或 unix:/run/php/php8.1-fpm.sock
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 日志
    access_log /var/log/nginx/swiftadmin-access.log;
    error_log /var/log/nginx/swiftadmin-error.log;
}
```

### 8.2 HTTPS 配置（推荐）

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    root /www/wwwroot/SwiftAdmin/public;
    index index.php index.html;

    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    # ... 其余配置同上 ...
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}
```

### 8.3 Apache 配置 .htaccess

如果使用 Apache，在 `public/` 目录下创建 `.htaccess`：

```apache
<IfModule mod_rewrite.c>
    Options +FollowSymlinks -Multiviews
    RewriteEngine On

    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^(.*)$ index.php?/$1 [QSA,PT,L]
</IfModule>
```

### 8.4 宝塔面板伪静态

在宝塔站点设置 > 伪静态 中粘贴：

```nginx
location / {
    if (!-e $request_filename) {
        rewrite ^(.*)$ /index.php?s=$1 last;
    }
}
```

## 9. 生产环境优化

### 9.1 PHP OPcache 加速

编辑 `php.ini`：

```ini
; 开启 OPcache
opcache.enable = 1
opcache.enable_cli = 1
opcache.memory_consumption = 128
opcache.interned_strings_buffer = 16
opcache.max_accelerated_files = 10000
opcache.validate_timestamps = 0
opcache.revalidate_freq = 0
opcache.save_comments = 0
```

### 9.2 Redis Session

修改 ThinkPHP Session 驱动为 Redis，在配置文件中添加：

```php
// config/session.php
return [
    'type' => 'redis',
    'host' => '127.0.0.1',
    'port' => 6379,
];
```

### 9.3 Composer 生产模式优化

```bash
# 只安装生产依赖，跳过开发依赖
composer install --no-dev --optimize-autoloader --classmap-authoritative

# 清理 Composer 缓存（可选）
composer clear-cache
```

### 9.4 MySQL 优化建议

```ini
# my.cnf
[mysqld]
innodb_buffer_pool_size = 1G         # 设为可用内存的 50%-70%
innodb_log_file_size = 256M
innodb_flush_log_at_trx_commit = 2   # 生产环境可设为 2（性能优先）
max_connections = 200
query_cache_type = 0                 # MySQL 5.7 建议关闭查询缓存
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
```

### 9.5 Nginx 优化建议

```nginx
# 开启 gzip 压缩
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;

# 启用 sendfile
sendfile on;
tcp_nopush on;
```

### 9.6 定时任务

建议添加系统 crontab 定期清理运行时文件：

```bash
# crontab -e
# 每天凌晨 3 点清理过期缓存
0 3 * * * /usr/bin/php /www/wwwroot/SwiftAdmin/think clear --cache >> /var/log/swiftadmin-clear.log 2>&1
```

### 9.7 安全加固清单

- [ ] `.env` 文件设置为 600 权限，禁止 Web 访问
- [ ] `APP_DEBUG` 设为 `false`
- [ ] 修改默认管理员密码 `admin/admin123`
- [ ] Nginx 禁止访问 `.env`、`.git`、`composer.json` 等敏感文件
- [ ] 配置防火墙，仅开放 80/443 端口
- [ ] 定期备份数据库 `mysqldump swiftadmin > backup.sql`
- [ ] 保持 PHP、MySQL、Nginx 版本更新

### 9.8 目录权限参考

| 目录 / 文件 | 权限 | 说明 |
|------------|------|------|
| `.env` | 600 | 含敏感信息，仅 root 可读写 |
| `runtime/` | 777 | ThinkPHP 运行时目录（缓存、日志） |
| `public/uploads/` | 777 | 用户上传文件 |
| `public/static/` | 777 | 静态资源（可写入的场景） |
| `vendor/` | 755 | Composer 依赖，无需写权限 |
| `composer.json` | 644 | 只读即可 |
| 其他 `.php` 文件 | 644 | 只读即可 |

## 10. 常见问题

### 10.1 页面 404 或样式丢失

**原因**: URL 重写未生效或站点根目录未指向 `public/`。

**解决方案**:
1. 确认 Nginx/Apache 站点根目录指向 `<项目根>/public/`
2. 确认伪静态（URL 重写）已正确配置
3. 检查前端文件是否已构建：`ls public/admin/` 应有 `index.html`

### 10.2 数据库连接失败

**错误信息**: `SQLSTATE[HY000] [1045] Access denied for user...`

**解决方案**:
1. 检查 `.env` 中数据库用户名、密码、主机名是否正确
2. 确认数据库用户有对应数据库的访问权限
3. Docker 环境下数据库主机名应为 `mysql`（容器服务名）

### 10.3 PHP 扩展缺失

**错误信息**: `Class 'Redis' not found` 或 `Call to undefined function imagecreate()`

**解决方案**:
```bash
# Linux (Ubuntu/Debian)
sudo apt install php-redis php-gd

# Linux (CentOS/RHEL)
sudo yum install php-redis php-gd

# macOS (Homebrew)
brew install php && brew link php

# 宝塔面板
# 软件商店 > PHP设置 > 安装扩展 > 勾选 redis / gd
```

### 10.4 composer install 失败

**原因**: 网络问题、PHP 版本过低、缺少扩展。

**解决方案**:
```bash
# 使用国内镜像源
composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/

# 确认 PHP 版本 >= 7.2.5
php -v

# 增加内存限制
php -d memory_limit=-1 $(which composer) install
```

### 10.5 npm install / npm run build 失败

**原因**: Node.js 版本过低、网络问题。

**解决方案**:
```bash
# 确认 Node.js 版本 >= 16
node -v

# 使用国内镜像
npm config set registry https://registry.npmmirror.com

# 清除缓存后重试
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### 10.6 上传文件失败

**原因**: 上传目录权限不足或 PHP 上传限制。

**解决方案**:
1. 确保 `public/uploads/` 目录权限为 777
2. 修改 `php.ini`:
```ini
upload_max_filesize = 50M
post_max_size = 55M
max_execution_time = 300
```

### 10.7 验证码不显示

**原因**: GD 扩展缺失或 Session 路径权限问题。

**解决方案**:
```bash
# 检查 GD 扩展
php -m | grep gd

# 确认 runtime/ 目录可写
chmod -R 777 runtime/
```

### 10.8 Docker 容器启动失败

**解决方案**:
```bash
# 查看具体错误
docker-compose logs mysql
docker-compose logs app
docker-compose logs nginx

# 端口冲突（如 3306 已被占用）时修改端口
NGINX_PORT=8080 DB_PORT=3307 docker-compose up -d

# 完全重建（清除旧数据和镜像）
docker-compose down -v
docker-compose up -d --build --force-recreate
```

### 10.9 宝塔面板 502 Bad Gateway

**解决方案**:
1. 确认 PHP 版本已安装并启动
2. 检查 PHP-FPM 是否运行：`systemctl status php-fpm-81`（版本号根据实际情况）
3. 检查站点 PHP 版本设置是否正确
4. 查看 PHP 错误日志：`/www/server/php/81/var/log/php-fpm.log`

### 10.10 后台登录后跳回登录页

**解决方案**:
1. 检查 Session 配置，确认 `runtime/` 目录可写
2. 如果使用 Redis Session，确认 Redis 服务正常运行
3. 清除浏览器缓存和 Cookie 后重试
4. 检查服务器时间是否正确
